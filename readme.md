## 前言概览

+ 这原本是大二下学期信号与系统的第一次大作业与大三下“空间观测技术 ”课程汇报的相关程序 

+ 原题是"采集一段声音,利用MATLAB对信号进行处理"(大二)与"自选题目汇报,涉及傅里叶变换、最小二乘拟合与卡尔曼滤波其一"（大三）,刚刚好信号处理逻辑与之前我设想的 **=="让盲人听见星空"==** 计划有契合之处,所以————这就是这个计划的起点   (~~所以ddl才是最大的源动力吗~~)

+ 图片转音频 :
   + 大体逻辑是将一张星空图片转为==灰度图==,然后按列转化为==高维数组==(就是numpy的思维),计算每列对应的 **样本数** 和每列的**平均亮度**后,**将亮度映射到频率,生成对应列的音频信号,并将生成的音频信号填充到音频数组中**
   +  看起来是可行的……达嘎,口头挖路！这个方法由于单纯将原本信息转为灰度图且只是将亮度映射到音频上去，这样会丢掉许多天体的信息
   +  对于专业的天文图片，可以将**RGB三个通道**分别转化为**不同的灰度图**,分别映射到**三个不同的频率带上**(最好隔得远一点), ~~然后再将各自的亮度转化为不同的音高~~(~~放屁,这样同频带的音量会相互掩盖的~~) 在三个不同的频率带上各自又将不同灰度图的数据的每一组亮度映射到**各自的频率带**上 ,三个高维数组按列**混频输出**,就能将三通道的图片转为高信息量的一段音频了(==可以把它理解为音频状态下的"伪色图"==)
   +  值得一提的是,numpy这种 ==按高维数组==遍历处理数据的模式有点类似于NASA在网站上按列扫描图片并转化为音频的方法(当然他们还做了类似于极坐标雷达扫描的方法,这个之后有时间再研究吧),因此用Python做这玩意有天生的优势 (~~MATLAB,xx都不用!~~)

   ## 正式介绍

   这个项目本意是希望用更多人体感官来用另一个视角带大家领略到不一样的宇宙，包括但不限于听觉、触觉、嗅觉等等，用熟悉的感官发现不一样的宇宙；BI9CEJ也希望借此契机来帮助更多视障人士来感知天文领域的信息，将缕缕看不见的星光以另一种形式呈现出来，能让更多想仰望星空却无法实现的朋友接触、感兴趣甚至热爱并参与到宇宙探索的进程中；目前BI9CEJ第一阶段的工作是天文听觉程序的开发,之后会渐渐过渡到更多感官，并搭建出相关硬件，用于实际的视障人士工作场景以及科普工作中（包括面向视障人士的公民科学项目）。

   #### 天文触觉部分

   <small>以下内容是我的灵感，所以没有太详细的分条整理</small>

   <small>从某种程度上，效果有点类似于project hail mary 中 Rocky的沙盘</small>

   把星图弄成可触摸化的，然后实时更新坐标；对于一些星等相差较大的天体，可以用对数函数来压缩，或者语音播报；然后对于天空也如此，可以通过摄像头将天体位置转化为可触摸的模式，然后可以“摸”遍整片天空；用很多根探针来模拟，然后探针可以通上微电流和加上温度，用于提供更多元化的信息

   在技术上是可行的，主控可以用一个树莓派4b或者5b，然后星图采集和处理用一个工控机，使用多串口通信来加快传输效率，控制“沙盘”可以从树莓派链接上ESP32、树莓派nano或者arduino nano（体积小、管脚多、性能高）；可以弄一个亚克力球，内部加上加热片，感应到的天区可以升温来提示用户

   甚至如果不是特殊天象的话，可以略过图像采集，只需要搞一个加速度传感器、陀螺仪就可以通过用户手柄配合我前面提到的亚克力球来控制指向的天区（同样的，盲人可以用这个方法操控赤道仪或者经纬仪），然后沙盘星图实时更新，这样就可以无视天气观测星空了

   沙盘的相对高度只能表征图像灰度图，所以加上温度和微电流和智能语音来表征多维信息

   颜色的话，可以把图像分为RGB通道，然后切换显示；至于距离的话，也可以通过语音播报来进行

   因为天体最重要的特征之一是不同波段下的形态，沙盘可以解决这个问题

   如果沙盘数据点比较多的话，可以用寄存器存储多进制信息，然后一行探针用一个管脚就可以

   （hhh 3D打印也不错，就是花时间，适合展示全向立体结构，而且好多人做过了来着）
